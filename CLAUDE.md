# 像素呼吸 — 视频选题工作区

这是「像素呼吸」YouTube/抖音频道的视频制作工作区。频道定位：10分钟认知提升类视频，用经济学 × AI认知的交叉视角讲故事。

## 脚本制作规范

### 核心规则

1. **必备产出文件**：口播逐字稿 + 口播逐字稿-剪映版 + 发布清单（放在同一个选题文件夹下）
2. **不生成分镜脚本**：本工作区当前不需要 `分镜脚本.md`，也不按旧规则强制产出分镜表格
3. **口播逐字稿**：中文 ≥ 2800字，一行一句，用 `## 第X章：章节标题` 分隔
4. **四章结构**：每章标题格式 `## 第X章：[标题]`，章节内可用一句 `本章核心` 辅助梳理
5. **视频 Prompt 后缀**：`Cinematic lighting, hyper-realistic, dark moody color palette, 8k resolution`

### 认知类视频顿悟感规则

每次生成新脚本前，先确定一个且只确定一个核心认知反转。整条视频只打穿这个点，不要像写文章一样铺太多信息。

认知类视频要少讲道理，多制造“顿悟感”。信息密度高不等于好看，用户需要的是一个明确爽点：听完以后觉得“原来我之前理解错了”。

每条视频优先使用以下反转句式之一作为主轴：

- 你以为 X，其实真正决定结果的是 Y
- 大多数人理解错了 X
- 这个现象背后不是情绪问题，是结构问题
- 普通人最该警惕的不是 A，而是 B

开头不要铺背景，直接给判断。前三句必须让用户觉得“这句话跟我有关，而且可能颠覆我”。

推荐开头句式：

- 我发现很多人对 [主题] 最大的误解是……
- 这件事最反直觉的地方在于……
- 普通人真正被拉开差距，不是因为 [表层原因]，而是因为……
- 如果你现在还这样理解 [短视频/赚钱/学习/关系/AI]，基本会越来越被动。

脚本自检：

- [ ] 这一条只有一个核心认知反转
- [ ] 前 3 秒没有铺背景，直接给判断
- [ ] 用户能在开头 10 秒内知道“我之前可能理解错了什么”
- [ ] 每章都服务同一个反转，不另开新课题
- [ ] 少解释概念，多用生活场景让用户自己顿悟
- [ ] 结尾回到这个反转，给一个普通人今天能用的判断

### 写作风格

- **先像人，再像知识博主**：口播要像一个懂行的朋友在饭桌上讲明白一件事，不像报告、课程、百科词条。
- **开头从具体动作切入**：优先写"你拧开瓶盖""你点外卖""你把盐撒进锅里"这种画面，少用"今天我们来讲一个概念"。
- **反常识要自然出现**：可以有悬念，但不要频繁使用"真相是""本质是""你以为，其实"这类模板句。
- **每章至少1次身份代入**：用真实生活处境代入，不要只机械写"你……"。例如"你月底看账单时的那一下心虚"，比"你会发现"更像人话。
- **每章开头要换气**：章节开头先给一个新画面、新冲突或新问题，避免总用"回到正题""注意听""讲到这里"。
- **数据必须有对比锚点**：数字后面要接生活尺度，例如"这相当于一个县城的人每天都在交这笔钱"。
- **金句宁少勿硬**：至少3句可传播表达，但不能写"金句来了"，也不要每个原理都强行总结。金句要从故事里长出来。
- **口语化短句**：一句话尽量不超过25字，但允许少量长句制造节奏变化。
- **保留人的犹豫和转折**：可以写"这听起来有点夸张""但你仔细想一下""问题就出在这里"，让稿子有思考过程。
- **少讲抽象词，多讲具体物**：每讲一个概念，必须落回一个人、一件东西、一个价格、一个场景。
- **禁用高频AI腔**：避免"深度赋能""认知升级""底层逻辑""显著提升""闭环""抓手""高维度""本质上""不难发现""值得注意的是""在这个时代"。
- **禁用成功学压迫感**：少用"普通人为什么不行""顶级高手""底层人""你必须""否则就会被淘汰"。
- **少用排比堆概念**：连续抽象名词不超过3个，经济学术语出现后要立刻翻译成人话。
- **收尾要有余味**：不喊口号，不催行动，用一个具体画面或一句轻轻落下来的判断结束。

### 文件夹结构

```
进行中/选题名/
├── 选题名-口播逐字稿.md
├── 选题名-口播逐字稿-剪映版.md    # 纯文本，一行一句，无标记
├── 选题名-双语字幕.srt           # 录制后生成
└── 选题名-发布清单.md
```

注意：不要额外创建 `选题名-分镜脚本.md`。如果需要剪辑素材方向，写进发布清单的“画面方向”即可。

### 发布清单规范

每次视频完成后必须产出发布清单，**参考模板：`模板/发布清单模板.md`**，也可参考已完成稿件里的发布清单。

清单必须包含以下板块：

1. **发布前检查**
   - 视频质量（时长、hook、字幕、音量、封面）
2. **发布设置**
   - 平台与账号确认
   - 3个推荐标题（含推荐理由）
   - 标签
   - 发布文案（描述）
   - 发布时间（首选/次选/保底）
3. **发布后**
   - 播放检查、评论互动、数据截图
4. **下一条视频**
   - 按选题库推荐下一期，确认结尾已埋预告钩子

### 双语字幕规范（SRT）

每条字幕必须包含两行文本，格式如下：

```
序号
时间轴
中文原文（去标点）
英文翻译（去标点）
（空行）
```

**翻译要求：**
- 第一行：中文原文，去掉所有标点符号（，。？！、；：等）
- 第二行：英文翻译，自然口语化，去掉句末标点，适合字幕阅读
- 英文保持简洁短句风格，与中文原意对应
- 专有名词保留原名（ChatGPT、Daniel Kahneman 等）
- 百分比、数字等保留原格式

**示例：**
```
1
00:00:00,166 --> 00:00:02,366
你有没有觉得AI特别懂你
Do you ever feel like AI really gets you
```
